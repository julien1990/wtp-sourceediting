434         * <code>null</code> will be returned.
435         * <p>
436         * <b>Note:</b> <code>@SuppressWarnings</code> can only suppress warnings,
437         * which means that if some problems got promoted to ERROR using custom compiler
438         * settings ({@link IJavaScriptProject#setOption(String, String)}), the
439         * <code>@SuppressWarnings</code> annotation will be ineffective.
440         * </p>
441         * <p>
442         * <b>Note:</b> <code>@SuppressWarnings</code> can be argumented with
443         * <code>"all"</code> so as to suppress all possible warnings at once.
444         * </p>
445         * <p>
446         * <b>Note:</b> The tokens returned are not necessarily standardized across JavaScript
447         * validators. If you were to use one of these tokens in an @SuppressWarnings
448         * annotation in the JavaScript source code, the effects (if any) may vary from
449         * validator to validator.
450         * </p>
451         * @param problemID
452         *         the ID of a given warning to suppress
453         * @return a String which can be used in <code>@SuppressWarnings</code> annotation,
454         * or <code>null</code> if unable to suppress this warning.
455         */
456        public static String getWarningToken(int problemID){
457                long irritant = ProblemReporter.getIrritant(problemID);
458                if (irritant != 0) {
459                        return CompilerOptions.warningTokenFromIrritant(irritant);
460                }
461                return null;
462        }
