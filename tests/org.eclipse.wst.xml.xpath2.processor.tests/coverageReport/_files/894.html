30/**
31 * Abstract base implementation of all includepath container initializer.
32 * Includepath variable containers are used in conjunction with the
33 * "org.eclipse.wst.jsdt.core.JsGlobalScopeContainerInitializer" extension point.
34 * <p>
35 * Clients should subclass this class to implement a specific includepath
36 * container initializer. The subclass must have a public 0-argument
37 * constructor and a concrete implementation of <code>initialize</code>.
38 * <p>
39 * Multiple includepath containers can be registered, each of them declares
40 * the container ID they can handle, so as to narrow the set of containers they
41 * can resolve, in other words, a container initializer is guaranteed to only be
42 * activated to resolve containers which match the ID they registered onto.
43 * <p>
44 * In case multiple container initializers collide on the same container ID, the first
45 * registered one will be invoked.

48 * @see IJsGlobalScopeContainer
