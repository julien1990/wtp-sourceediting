388     * Defaults to an entire javaScript unit.

391         * requested source:
392         * <ul>
393         * <li>{@link #K_JAVASCRIPT_UNIT K_JAVASCRIPT_UNIT}: The result node
394         * is a {@link JavaScriptUnit}.</li>
395         * <li>{@link #K_CLASS_BODY_DECLARATIONS K_CLASS_BODY_DECLARATIONS}: The result node

397         * {@link TypeDeclaration#bodyDeclarations() bodyDeclarations}

401         * are the new trees. Other aspects of the block are unspecified.</li>
402         * <li>{@link #K_EXPRESSION K_EXPRESSION}: The result node is a subclass of

405         * The resulting AST node is rooted under (possibly contrived)
406         * {@link JavaScriptUnit JavaScriptUnit} node, to allow the
407         * client to retrieve the following pieces of information
408         * available there:
409         * <ul>

411         * numbers start at 1 and only cover the subrange scanned

415         * Character positions are relative to the start of
416         * <code>source</code>; line positions are for the subrange scanned.</li>
417         * <li>{@linkplain JavaScriptUnit#getCommentList() Comment list}
418         * for the subrange scanned.</li>
419         * </ul>
420         * The contrived nodes do not have source positions. Other aspects of the
421         * {@link JavaScriptUnit JavaScriptUnit} node are unspecified, including
422         * the exact arrangment of intervening nodes.
423         * </p>
424         * <p>

426         * a result node being marked as {@link ASTNode#MALFORMED MALFORMED}.

428         * recognize the input, this method returns
429         * a {@link JavaScriptUnit JavaScriptUnit} node with at least the
430         * validator messages.
431         * </p>
432         * <p>Each node in the subtree (other than the contrived nodes)
433         * carries source range(s) information relating back
434         * to positions in the given source (the given source itself
435         * is not remembered with the AST).
436         * The source range usually begins at the first character of the first token
437         * corresponding to the node; leading whitespace and comments are <b>not</b>
438         * included. The source range usually extends through the last character of
439         * the last token corresponding to the node; trailing whitespace and
440         * comments are <b>not</b> included. There are a handful of exceptions
441         * (including the various body declarations); the
442         * specification for these node type spells out the details.
443         * Source ranges nest properly: the source range for a child is always
444         * within the source range of its parent, and the source ranges of sibling
445         * nodes never overlap.
446         * </p>
447         * <p>
448         * Binding information is only computed when <code>kind</code> is
449     * <code>K_JAVASCRIPT_UNIT</code>.
450         * </p>
451         *
452         * @param kind the kind of construct to parse: one of
453         * {@link #K_JAVASCRIPT_UNIT},
454         * {@link #K_CLASS_BODY_DECLARATIONS},
455         * {@link #K_EXPRESSION},
456         * {@link #K_STATEMENTS}
457         */

465                this.astKind = kind;
466        }

656        /**
657     * Creates ASTs for a batch of javaScript units.
658     * When bindings are being resolved, processing a
659     * batch of javaScript units is more efficient because much
660     * of the work involved in resolving bindings can be shared.
661     * <p>
662     * When bindings are being resolved, all javaScript units must
663     * come from the same JavaScript project, which must be set beforehand
664     * with <code>setProject</code>.
665     * The javaScript units are processed one at a time in no
666     * specified order. For each of the javaScript units in turn,
667         * <ul>
668         * <li><code>ASTParser.createAST</code> is called to parse it
669         * and create a corresponding AST. The calls to
670         * <code>ASTParser.createAST</code> all employ the same settings.</li>
671         * <li><code>ASTRequestor.acceptAST</code> is called passing
672         * the javaScript unit and the corresponding AST to
673         * <code>requestor</code>.
674         * </li>
675         * </ul>
676     * Note only ASTs from the given javaScript units are reported
677     * to the requestor. If additional javaScript units are required to
678     * resolve the original ones, the corresponding ASTs are <b>not</b>
679     * reported to the requestor.
680     * </p>
681         * <p>
682         * Note also the following parser parameters are used, regardless of what
683         * may have been specified:
684         * <ul>
685         * <li>The {@linkplain #setKind(int) parser kind} is <code>K_JAVASCRIPT_UNIT</code></li>
686         * <li>The {@linkplain #setSourceRange(int,int) source range} is <code>(0, -1)</code></li>
687         * <li>The {@linkplain #setFocalPosition(int) focal position} is not set</li>
688         * </ul>
689         * </p>
690     * <p>
691     * The <code>bindingKeys</code> parameter specifies bindings keys
692     * ({@link IBinding#getKey()}) that are to be looked up. These keys may
693     * be for elements either inside or outside the set of compilation
694     * units being processed. When bindings are being resolved,
695     * the keys and corresponding bindings (or <code>null</code> if none) are
696     * passed to <code>ASTRequestor.acceptBinding</code>. Note that binding keys
697     * for elements outside the set of javaScript units being processed are looked up
698     * after all <code>ASTRequestor.acceptAST</code> callbacks have been made.
699     * Binding keys for elements inside the set of javaScript units being processed
700     * are looked up and reported right after the corresponding
701     * <code>ASTRequestor.acceptAST</code> callback has been made.
702     * No <code>ASTRequestor.acceptBinding</code> callbacks are made unless
703     * bindings are being resolved.
704     * </p>
705     * <p>
706     * A successful call to this method returns all settings to their
707     * default values so the object is ready to be reused.
708     * </p>
709     *
710     * @param compilationUnits the javaScript units to create ASTs for
711     * @param bindingKeys the binding keys to create bindings for
712     * @param requestor the AST requestor that collects abtract syntax trees and bindings
713         * @param monitor the progress monitor used to report progress and request cancelation,
714         *   or <code>null</code> if none
715         * @exception IllegalStateException if the settings provided
716         * are insufficient, contradictory, or otherwise unsupported
717     */
718        public void createASTs(IJavaScriptUnit[] compilationUnits, String[] bindingKeys, ASTRequestor requestor, IProgressMonitor monitor) {
719                try {
720                        int flags = 0;
721                        if (this.statementsRecovery) flags |= IJavaScriptUnit.ENABLE_STATEMENTS_RECOVERY;
722                        if (this.resolveBindings) {
723                                if (this.project == null)
724                                        throw new IllegalStateException("project not specified"); //$NON-NLS-1$
725                                if (this.bindingsRecovery) flags |= IJavaScriptUnit.ENABLE_BINDINGS_RECOVERY;
726                                JavaScriptUnitResolver.resolve(compilationUnits, bindingKeys, requestor, this.apiLevel, this.compilerOptions, this.project, this.workingCopyOwner, flags, monitor);
727                        } else {
728                                JavaScriptUnitResolver.parse(compilationUnits, requestor, this.apiLevel, this.compilerOptions, flags, monitor);
729                        }
730                } finally {
731                      // re-init defaults to allow reuse (and avoid leaking)
732                      initializeDefaults();
733                }
734        }

929         * client to retrieve the following pieces of information
930         * available there:
931         * <ul>

934         * (<code>source[offset]</code> through <code>source[offset+length-1]</code>).</li>
935         * <li>{@linkplain JavaScriptUnit#getMessages() Compiler messages}
936         * and {@linkplain JavaScriptUnit#getProblems() detailed problem reports}.
937         * Character positions are relative to the start of
938         * <code>source</code>; line positions are for the subrange scanned.</li>
939         * <li>{@linkplain JavaScriptUnit#getCommentList() Comment list}
940         * for the subrange scanned.</li>

942         * The contrived nodes do not have source positions. Other aspects of the
943         * {@link JavaScriptUnit JavaScriptUnit} node are unspecified, including
944         * the exact arrangment of intervening nodes.
945         * </p>
946         * <p>

950         * recognize the input, this method returns

952         * validator messages.
953         * </p>

966         * within the source range of its parent, and the source ranges of sibling
967         * nodes never overlap.
968         * </p>
969         * <p>

971         * methods applied to nodes of the resulting AST return <code>null</code>.
972         * </p>
