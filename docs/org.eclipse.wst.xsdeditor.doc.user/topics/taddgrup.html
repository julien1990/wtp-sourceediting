Adding a group

A group consists of element declarations, wildcards, and other content model groups. When you create a group, it automatically contains a content model.

The following instructions were written for the Resource perspective, but they will also work in many other perspectives. 

To add a group, follow these steps:

  1. Open your XML schema in the XML schema editor.
  2. In the Outline view, select your schema. The entire schema and its contents should be displayed in the Graph view.
  3. In the Graph view, right click in the Groups section and click Add Group.
  4. In the Properties view, type a new name for the group in the Name field.
  5. Click the Documentation tab if you want to provide any information about this group element. The Documentation page is used for human readable material, such as a description, and the App Info page can be used to provide information for applications. 
  6. In the Outline view, expand the Groups folder and your new group.
  7. Your group automatically contains a content model. A group's content model is the representation of any data that may be contained inside the group. For more information about working with content models, refer to the related tasks.
