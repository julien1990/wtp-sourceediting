XSLT Processor

Identifier:
org.eclipse.wst.xsl.launching.processor

Description:
Contribute a Java XSLT processor instance (e.g. Xalan 2.7.0). The extension point primarily defines the classpath to be used when launching an XSLT transformation.

Configuration Markup:

<!ELEMENT extension (processor+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT processor EMPTY>

<!ATTLIST processor

id              CDATA #REQUIRED

label           CDATA #REQUIRED

processorTypeId CDATA #REQUIRED

classpath       CDATA #REQUIRED

debuggerId      CDATA #IMPLIED

supports        CDATA #REQUIRED>

  • id -
  • label -
  • processorTypeId -
  • classpath -
  • debuggerId -
  • supports - A comma-separated list of supported XSLT versions e.g. 1.0,2.0

Examples: 

   <extension
         point="org.eclipse.wst.xsl.launching.processor">
      <processor
            classpath="xalan2.7.0.jar;serializer2.7.0.jar"
            id="org.eclipse.wst.xsl.launching.xalan.2_7_0"
            label="Xalan 2.7.0"
            processorTypeId="org.eclipse.wst.xsl.xalan.processorType"
            debuggerId="org.eclipse.wst.xsl.xalan.debugger"
            supports="1.0">
      </processor>
   </extension>
